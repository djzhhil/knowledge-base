## [2026-03-05] Multi-Agent 配置分析
- 因：耀的三个 agent（main, dev-agent, browser-agent）配置集中在 AGENTS.md，其他两个 agent 的配置文件为空
- 改：阅读 OpenClaw 官方文档，了解官方推荐的多 agent 使用方式
- 官方推荐：
  - Multi-Agent Routing：多个独立 agent，每个有独立 workspace、SOUL.md、AGENT.md
  - Sub-Agents：临时子会话，通过 sessions_spawn 创建，完成后自动销毁
- 结论：当前架构混合了两种方式，按照官方推荐应该用 Multiple Agent Routing
