## [2026-03-06 20-21点] Summarize 工具配置修复成功
- **问题**：summarize 配置不正确导致无法使用
- **错误配置**：
  ```json
  {
    "model": "openai/llama-3.3-70b-versatile",
    "apiBase": "https://api.groq.com/openai/v1"  // ❌ 不支持
  }
  ```
- **正确配置**（按官方文档）：
  ```json
  {
    "model": "openai/llama-3.3-70b-versatile",
    "env": {
      "OPENAI_API_KEY": "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",
      "OPENAI_BASE_URL": "https://api.groq.com/openai/v1"
    }
  }
  ```
- **关键发现**：
  - ❌ 不支持 `apiBase` 配置项
  - ✅ 使用 `env` 键存储环境变量
  - ✅ OpenAI 兼容端点使用 `OPENAI_BASE_URL`
- **方案对比**：
  - 方案 1：Gemini API（需代理，Node.js fetch 自动不使用代理）
  - 方案 2：Groq API（✅ 推荐，无地域限制，速度快，免费）
- **环境变量修复**：`UVX_PATH=$HOME/.local/bin/uvx`（uvx 已安装但不在 PATH）
- **PDF 读取**：
  - `--extract-only` 只支持媒体文件（MP3/MP4/WAV），不支持 PDF
  - `--length` 控制总结长度（xs/s/m/l/xl），不是提取完整内容
  - PDF 转换依赖 `uvx markitdown`（首次运行会下载依赖）
