## 📋 需求分析

### 现有系统现状

- **技术栈：** Vue 3 + Spring Boot + MySQL
- **现有功能：**
  - 笔记增删改查
  - Markdown 支持（仅文本输入）
  - 分类管理
  - 标签系统
  - 搜索功能
  - 按分类筛选
- **当前 UI：** 简单的卡片式布局，原生 CSS，无预览功能

### 新需求

1. **UI 重新设计** - 参考 Obsidian、Notion 等现代笔记软件
   - 需要更现代、美观、易用的界面
   - 支持实时 Markdown 预览
   - 优化布局和交互体验

2. **实现 MD 文档上传功能**
   - 支持上传本地 .md 文件
   - 自动解析并导入为笔记
   - 支持批量上传

3. **推送到 GitHub**
   - 将改造后的代码推送到 GitHub 仓库

## 📝 开发注意事项

### 前端开发
1. **保持组件化思维：** 将大组件拆分为小组件（NoteCard、NoteList等）
2. **状态管理：** 考虑使用 Pinia 管理全局状态（主题、用户偏好）
3. **响应式设计：** 使用 Element Plus 的响应式栅格系统
4. **错误处理：** 使用 try-catch 和 Element Plus 的 ElMessage 统一提示

### 后端开发
1. **文件安全性：** 校验文件类型、大小，防止恶意文件上传
2. **异常处理：** 统一全局异常处理器
3. **日志记录：** 记录文件上传和导入的详细日志
4. **并发处理：** 批量导入时考虑使用线程池

### Git 规范
- 提交信息使用 Conventional Commits 规范：
  - `feat:` 新功能
  - `fix:` Bug 修复
  - `style:` 样式调整
  - `docs:` 文档更新
  - `refactor:` 代码重构

## 🚀 快速启动命令

### 前端依赖安装
```bash
cd frontend
npm install element-plus @element-plus/icons-vue mavon-editor
```

### 后端依赖添加
```xml
<!-- pom.xml -->
<dependency>
    <groupId>commons-io</groupId>
    <artifactId>commons-io</artifactId>
    <version>2.11.0</version>
</dependency>
```

### 启动项目
```bash
# 后端
cd backend
mvn spring-boot:run

# 前端（新终端）
cd frontend
npm run serve
```
