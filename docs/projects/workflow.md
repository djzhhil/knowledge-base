# 软件开发工作流程

## 完整开发流程

### 阶段 1：需求分析 (PM Agent)
```
1. 接收用户想法/需求
2. 分析需求可行性
3. 拆解需求为具体功能点
4. 输出：需求文档 (PRD)
```

### 阶段 2：设计规划 (Design Agent)
```
1. 分析需求文档
2. 设计系统架构
3. 设计数据库结构
4. 设计 API 接口
5. 设计 UI/UX 原型
6. 输出：设计文档 + API 文档
```

### 阶段 3：代码实现 (Dev Agent)
```
1. 根据设计文档编写代码
2. 按照开发规范编写
3. 代码自测
4. 输出：可运行代码
```

### 阶段 4：质量保证 (Test Agent)
```
1. 编写测试用例
2. 执行功能测试
3. 性能测试
4. 边界测试
5. 输出：测试报告
```

### 阶段 5：代码审查 (Review Agent)
```
1. 代码风格检查
2. 安全性检查
3. 性能检查
4. 最佳实践检查
5. 输出：审查报告
```

### 阶段 6：部署上线 (DevOps Agent)
```
1. 准备部署环境
2. 执行部署
3. 健康检查
4. 输出：部署报告
```

### 阶段 7：验收 (QA Agent)
```
1. 对照需求验收功能
2. 测试实际使用场景
3. 记录问题
4. 输出：验收报告
```

---

## Agent 协作矩阵

| 阶段 | 主责 Agent | 协作 Agent |
|------|-----------|-----------|
| 1. 需求分析 | pm-agent | - |
| 2. 设计规划 | design-agent | pm-agent, dev-agent |
| 3. 代码实现 | dev-agent | design-agent |
| 4. 质量保证 | test-agent | dev-agent |
| 5. 代码审查 | review-agent | dev-agent, test-agent |
| 6. 部署上线 | devops-agent | test-agent |
| 7. 验收 | qa-agent | 所有 agent |

---

## 标准交付物

### 1. 需求文档 (PRD)
- 功能描述
- 用户故事
- 非功能性需求
- 验收标准

### 2. 设计文档
- 系统架构图
- 数据库设计 ER 图
- API 接口文档
- UI/UX 原型

### 3. 代码交付
- 源代码（符合规范）
- 单元测试
- 接口文档

### 4. 测试报告
- 测试用例
- 测试结果
- Bug 列表
- 性能数据

### 5. 审查报告
- 代码问题
- 改进建议
- 风险提示

### 6. 部署报告
- 部署步骤
- 环境配置
- 健康检查结果

### 7. 验收报告
- 功能验收结果
- 使用体验
- 待优化项

---

## 快速迭代流程 (小功能)

对于小功能，可以简化流程：

```
用户想法 → PM Agent (分析) → Dev Agent (实现) → Test Agent (测试) → 部署
```
